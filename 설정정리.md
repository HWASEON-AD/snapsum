# SnapSum 설정 정리

> 마지막 업데이트: 2026-06-01

## 기술 스택
- **Framework**: Next.js (App Router) + TypeScript + Tailwind CSS
- **AI**: Claude Vision (claude-sonnet-4-6) + Prompt Caching
- **DB**: Supabase (items 테이블)
- **배포**: Vercel
- **PWA**: manifest.json + sw.js (Service Worker)

## 실행 방법
```bash
cd C:\Users\gtmin\Dropbox\5.개발\snapsum
npm run dev        # 개발 서버 (localhost:3000)
npm run build      # 프로덕션 빌드
```

## 환경변수 (.env.local)
```env
ANTHROPIC_API_KEY=           # Claude Vision API (필수)
NEXT_PUBLIC_SUPABASE_URL=    # Supabase 프로젝트 URL (필수)
NEXT_PUBLIC_SUPABASE_ANON_KEY=  # Supabase anon key (필수)
SUPABASE_SERVICE_ROLE_KEY=   # Supabase service role (서버 전용, 필수)
SNAPSUM_API_KEY=             # iOS/Android 클라이언트 인증 키 (필수)
```
현재 개발용 키: `SNAPSUM_API_KEY=snapsum-dev-key-2026`

## Supabase 테이블 SQL
```sql
CREATE TABLE items (
  id              UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  user_id         TEXT NOT NULL,
  created_at      TIMESTAMPTZ NOT NULL DEFAULT NOW(),
  source_app      TEXT NOT NULL,
  content_type    TEXT NOT NULL CHECK (content_type IN ('shopping','news','video','article')),
  title           TEXT NOT NULL,
  url             TEXT,
  thumbnail_url   TEXT,
  analysis        JSONB NOT NULL DEFAULT '{}',
  tags            TEXT[] NOT NULL DEFAULT '{}',
  summary_line    TEXT NOT NULL DEFAULT '',
  raw_screenshot_url TEXT
);
CREATE INDEX idx_items_user_id ON items (user_id);
CREATE INDEX idx_items_content_type ON items (content_type);
CREATE INDEX idx_items_created_at ON items (created_at DESC);
```
→ https://supabase.com/dashboard 에서 SQL Editor로 실행

## API 엔드포인트
| 엔드포인트 | 메서드 | 설명 | 인증 |
|-----------|--------|------|------|
| /api/analyze | POST | Claude Vision 분석 + 저장 (iOS/Android용) | x-api-key 헤더 |
| /api/upload | POST | 웹 업로드 분석 + 저장 (PWA용) | 없음 |
| /api/items | GET | 목록 조회 (?user_id=&type=&page=&limit=) | x-api-key 헤더 |
| /api/items/[id] | GET | 상세 조회 | x-api-key 헤더 |
| /api/items/[id] | DELETE | 삭제 | x-api-key 헤더 |

## iOS Shortcut 설정 방법
1. iPhone → 단축어 앱 → 자동화 → 새 자동화
2. 트리거: "스크린샷을 찍을 때"
3. 액션 추가:
   - 최신 사진 가져오기 (1장)
   - Base64로 인코딩
   - URL 콘텐츠 가져오기:
     - URL: https://snapsum.vercel.app/api/analyze
     - 방법: POST
     - 헤더: x-api-key: {SNAPSUM_API_KEY}
     - 본문: {"image":"{Base64}","image_type":"image/jpeg","user_id":"default"}
   - 알림 표시: "Saved to SnapSum ✓"
4. "실행 전에 묻기" → 끄기
